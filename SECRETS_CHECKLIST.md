## Replit-Specific Variables to Replace

These are auto-generated by Replit and won't exist on other platforms:

```bash
# Auto-generated by Replit (DO NOT COPY)
REPL_OWNER=[automatically set]
REPL_ID=[automatically set]
REPLIT_DEV_DOMAIN=[automatically set]
HOME=[automatically set]
```

**What to do**: 
- Remove code that depends on `REPL_ID`, `REPL_OWNER`
- Replace `REPLIT_DEV_DOMAIN` with your actual domain
- `HOME` will be set automatically by your new host

## How to Extract Secrets from Replit

### Method 1: Manual Copy (Safest)

1. Open Replit Secrets tool (lock icon in left sidebar)
2. For each secret listed above:
   - Click to reveal the value
   - Copy to a secure password manager
   - Paste into new hosting provider's environment variables

### Method 2: Via Code (Automated)

Create a temporary script in Replit Shell:

```javascript
// secrets-export.cjs
console.log('=== MySeniorValet Secrets Export ===\n');

const secrets = [
  'DATABASE_URL',
  'PGHOST', 'PGUSER', 'PGPASSWORD', 'PGDATABASE', 'PGPORT',
  'PERPLEXITY_API_KEY', 'ANTHROPIC_API_KEY', 'OPENAI_API_KEY',
  'SESSION_SECRET', 'JWT_SECRET',
  'GOOGLE_CLIENT_ID', 'GOOGLE_CLIENT_SECRET',
  'STRIPE_SECRET_KEY', 'STRIPE_WEBHOOK_SECRET',
  'SENDGRID_API_KEY',
  'REPLIT_DB_URL'
];

secrets.forEach(key => {
  const value = process.env[key];
  if (value) {
    console.log(`${key}=${value}`);
  } else {
    console.log(`${key}=(not set)`);
  }
});

console.log('\n=== End of Export ===');
```

Run it:
```bash
node secrets-export.cjs > secrets-backup.txt
```

**⚠️ SECURITY WARNING**: 
- Delete `secrets-backup.txt` immediately after copying
- Never commit this file to git
- Don't share it publicly

## Security Best Practices

### Do's ✅
- ✅ Use a password manager for secrets
- ✅ Rotate secrets after migration
- ✅ Use different secrets for dev/staging/prod
- ✅ Encrypt secrets at rest
- ✅ Limit access to secrets
- ✅ Enable 2FA on all service accounts
- ✅ Set up secret rotation policies

### Don'ts ❌
- ❌ Never commit secrets to git
- ❌ Never log secrets to console
- ❌ Never send secrets via email/Slack
- ❌ Never reuse secrets across projects
- ❌ Never share secrets in screenshots
- ❌ Never store secrets in plain text files
- ❌ Never use production secrets in development

## Emergency Secret Rotation

If secrets are compromised, rotate immediately in this order:

### Priority 1 (Rotate Within 1 Hour)
1. **DATABASE_URL / PGPASSWORD** - Highest risk, immediate data access
2. **STRIPE_SECRET_KEY** - Financial data at risk
3. **JWT_SECRET / SESSION_SECRET** - All user sessions compromised

### Priority 2 (Rotate Within 24 Hours)
4. **API Keys** (Perplexity, Anthropic, OpenAI, SendGrid)
5. **OAuth Secrets** (Google Client Secret)

### Priority 3 (Rotate Within 1 Week)
6. **STRIPE_WEBHOOK_SECRET** - Can be rotated with webhook URL update
7. **OAuth Client IDs** - Lower risk, but still rotate

## Service-Specific Notes

### Database Credentials
- **Where to find**: Replit Database tool → Connection Details
- **Format**: Full connection string or individual components
- **Rotation**: Change password in database provider settings

### Stripe Keys
- **Where to find**: https://dashboard.stripe.com/apikeys
- **Test vs Live**: Ensure you're using the correct environment keys
- **Webhook Secret**: Found in https://dashboard.stripe.com/webhooks

### Google OAuth
- **Where to find**: https://console.cloud.google.com/apis/credentials
- **Important**: Update redirect URIs when changing hosting
- **Testing**: Use OAuth Playground to test

### SendGrid
- **Where to find**: https://app.sendgrid.com/settings/api_keys
- **Permissions**: Ensure API key has "Mail Send" permission
- **Domain Auth**: Verify your sending domain

**🔒 Security First**: Treat secrets like passwords - never share, always encrypt, rotate regularly.

**📝 Document Everything**: Keep a secure record of where each secret is stored and what it's used for.

**🚨 Monitor Closely**: Watch logs for authentication errors or API failures that might indicate a missing or incorrect secret.
