✅ PART 1: Enable PostGIS in PostgreSQL

🛠 If you’re self-hosting (or local):

CREATE EXTENSION postgis;

If you’re using a hosted service (like Supabase, Neon, RDS), enable it in their PostGIS or Extensions tab.


---

✅ PART 2: Add location column to communities table

🧱 Migration SQL (Drizzle-compatible)

ALTER TABLE communities
ADD COLUMN location geography(Point, 4326);

UPDATE communities
SET location = ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)
WHERE latitude IS NOT NULL AND longitude IS NOT NULL;

CREATE INDEX idx_communities_location
ON communities
USING GIST (location);

✅ PART 3: Update Your Drizzle Schema

In shared/schema.ts, add:

import { pgTable, serial, text, doublePrecision, customType } from "drizzle-orm/pg-core";

const geographyPoint = customType<{ data: { lat: number; lng: number } }>({
  dataType() {
    return 'geography(Point,4326)';
  },
});

export const communities = pgTable('communities', {
  id: serial('id').primaryKey(),
  name: text('name').notNull(),
  // ... other fields
  latitude: doublePrecision('latitude'),
  longitude: doublePrecision('longitude'),
  location: geographyPoint('location'),
});

✅ PART 4: Update Search Endpoint in server/routes.ts

Add bounding box support:

app.get("/api/communities/search", async (req, res) => {
  const limit = parseInt(req.query.limit as string) || 300;
  const swLat = parseFloat(req.query.swLat as string);
  const swLng = parseFloat(req.query.swLng as string);
  const neLat = parseFloat(req.query.neLat as string);
  const neLng = parseFloat(req.query.neLng as string);

  try {
    const communities = await db.execute(sql`
      SELECT *
      FROM communities
      WHERE ST_Within(
        location,
        ST_MakeEnvelope(${swLng}, ${swLat}, ${neLng}, ${neLat}, 4326)
      )
      LIMIT ${limit};
    `);
    
    res.json(communities);
  } catch (err) {
    console.error("Geo search failed:", err);
    res.status(500).json({ error: "Geo search failed" });
  }
});
